Mental simulation¶

In other cases, we might use what's known as "mental simulation". This is essentially our ability to imagine objects, and manipulations of those objects, as if they were real.

For example, I can ask you if, when these two gears are rotated, if the bump will go in the notch. Most people will solve this problem by imagining the gears turning.

Similarly, if I show you this table with blocks on it and tell you that the table is bumped, you can tell me whether the yellow blocks are more likely to fall off, or the red blocks are more likely to fall off, by mentally simulating the outcome of different bumps.

Mental simulation¶

Shepard & Metzler (1971)

In the most famous demonstration of mental simulation, Shepard & Metzler asked people to compare images of objects, and determine whether those images showed the same object (which could be rotated), or different objects.

They famously found that the response time increased linearly with the angle of rotation between the objects. The conclusion is that people are "mentally rotating" the objects and then comparing them in order to determine if they are the same.

Mental simulation is a tool.¶

How should it be used?¶

Simulation is interesting, in that it is a complex and dynamic process that is potentially applicable to a wide range of scenarios.

So this leads to the question -- if mental simulation is a tool for us to use when solving problems, how should it be used?

A rational analysis of mental rotation¶

  • Problem: determine what spatial transformation(s) an object has undergone, given two images of that object
  • Assumption: we have a tool (mental simulation) that we can use to visualize transformations of an object (e.g., rotation, reflection)
  • Goal: solve the problem using the fewest number of mental simulations
  • Approach: choose to run simulations that will give the most information about the answer, so we can get away with fewer simulations

Models of mental rotation

  • Oracle model: computes the minimum angle of rotation, then rotates to it
  • Threshold model: chooses rotations until the transformed image looks "similar enough" to the original
  • Hill-climbing model: chooses rotations until it seems like rotating more would cause the images to look less similar
  • Bayesian quadrature model: actively chooses rotations which will provide the most information, until it is confident enough in its decision
    • unlike other models, tries to estimate the actual similarity function
    • chooses rotations which decrease the variance of this estimate
    • can also incorporate a bias towards different hypotheses through a prior

Summary¶

  • How should mental rotation be used?
  • Compared four models (oracle, threshold, hill climbing, Bayesian quadrature) to human data
  • On the surface, traditional models of mental rotation capture overall behavior, but they fail to account for more specific aspects of behavior
  • The active sampling model, which decides online which rotations to perform, can capture these behaviors to some extent
  • Active sampling thus seems promising in it's ability to help answer the question of "what to simulate?"

More generally...¶

  • Mental simulation is a tool for solving problems
  • So, models of mental simulation need to specify how that tool should be used
  • In the more general case, there are many questions that could be asked about how agents with multiple computational capacities can best make use of those abilities:
    • Should a simulation be used, or something else? (e.g. heuristics, logic, etc.)
    • What are the relevant things that need to be simulated?
    • How many simulations should be run?
    • How long should each simulation be run?
